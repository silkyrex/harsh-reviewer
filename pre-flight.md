# Pre-Flight Checklist

Before making code changes, do this quick check:

1. State the exact constraint in one sentence.
2. Extend existing code first; only add new structure if extension fails.
3. Pick the smallest change that solves the real problem.
4. Add or update a test that would catch the issue first.
5. Check error paths so failures are handled clearly.
6. Verify the change is easier to maintain than before.
